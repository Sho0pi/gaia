# Contributing to Gaia

Thanks for your interest! Gaia is an AI agent built on [google-adk](https://github.com/google/adk-python).
This guide covers the dev setup and the bar a change has to clear.

## Setup

Gaia uses [uv](https://docs.astral.sh/uv/) (never pip / poetry / venv by hand):

```bash
git clone https://github.com/Sho0pi/gaia.git
cd gaia
uv sync --all-groups
```

## The gate (run before every push)

A change isn't done until all four are green:

```bash
uv run ruff check --fix .      # lint
uv run ruff format .           # format
uv run mypy src                # types (strict)
uv run pytest                  # tests
```

CI runs the same on every PR (`check` + `system` jobs) and **must pass before merge**.

## Tests — what goes where

- **`tests/unit/`** — offline, fake/monkeypatch-driven, no keys, no network, fast. Every change
  ships these.
- **`tests/system/`** — touches something real: a model key (skipif-gated on `GEMINI_API_KEY`), a
  native lib (`importorskip`), or a live resource. Guards keep CI green without secrets — never
  remove them. The no-secret subset runs on every PR; the full set runs nightly.

A feature isn't done until both tiers exist and pass.

## Workflow

1. **Branch first** — never commit to `master` (it's protected; direct pushes are rejected).
2. **Match the library's idiom** — read ADK/A2A patterns and write in their style; prefer their
   public API. Reuse a proven library over reinventing.
3. **Open a PR** using the template — what & why, how, test evidence, and a short self-review
   (what's weak / what you'd improve next).
4. CI must be green; then the PR can merge.

## Where things live

- `src/gaia/` — the package (core orchestrator, souls, memory, missions, connectors, tools, cli).
- `docs/` — the [Astro Starlight](https://starlight.astro.build) docs site
  ([docs.gaia-agent.com](https://docs.gaia-agent.com)); its Reference pages are generated by
  `scripts/gen_reference.py`.
- `AGENTS.md` — the in-depth architecture + conventions (the project's source of truth for *how* it
  is built).

## Reporting bugs / requesting features

Use the [issue templates](https://github.com/Sho0pi/gaia/issues/new/choose). For security issues,
see [SECURITY.md](SECURITY.md) — please don't open a public issue.
